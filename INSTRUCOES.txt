━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

-------- INÍCIO DO SQL --------

-- 1. Tabela de perfis de usuário
CREATE TABLE IF NOT EXISTS profiles (
  id            UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  nome          TEXT,
  email         TEXT,
  role          TEXT NOT NULL DEFAULT 'user',
  licenca_id    UUID,
  created_at    TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 2. Tabela de licenças
CREATE TABLE IF NOT EXISTS licencas (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  codigo            TEXT NOT NULL UNIQUE,
  nome              TEXT NOT NULL,
  tipo              TEXT NOT NULL DEFAULT 'comprado',
  validade_ate      DATE,
  limite_familias   INTEGER NOT NULL DEFAULT 0,
  familias_ativas   INTEGER NOT NULL DEFAULT 0,
  ativa             BOOLEAN NOT NULL DEFAULT TRUE,
  observacao        TEXT DEFAULT '',
  licenca_admin_id  UUID,
  created_at        TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. Tabela de progresso de leitura
CREATE TABLE IF NOT EXISTS progresso (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  story_id    INTEGER NOT NULL,
  lida        BOOLEAN NOT NULL DEFAULT TRUE,
  updated_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(user_id, story_id)
);

-- 4. Tabela de convites
CREATE TABLE IF NOT EXISTS convites (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT NOT NULL,
  licenca_id  UUID NOT NULL REFERENCES licencas(id) ON DELETE CASCADE,
  token       TEXT NOT NULL UNIQUE,
  expira_em   TIMESTAMP WITH TIME ZONE NOT NULL,
  usado       BOOLEAN NOT NULL DEFAULT FALSE,
  usado_por   UUID,
  usado_em    TIMESTAMP WITH TIME ZONE,
  criado_por  UUID,
  created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 5. Tabela de configurações dinâmicas
CREATE TABLE IF NOT EXISTS configuracoes (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  chave      TEXT NOT NULL UNIQUE,
  valor      TEXT NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 6. Inserções padrão de configurações
INSERT INTO configuracoes (chave, valor) VALUES
  ('whatsapp_numero',        '5544984373004'),
  ('whatsapp_msg_visitante', 'Olá! Tenho uma dúvida sobre o Palavra Viva.'),
  ('whatsapp_msg_cliente',   'Olá! Sou assinante do Palavra Viva e preciso de ajuda.'),
  ('historias_destaque',     '1,10,20,30,50,80,120,200,300,400,500,700'),
  ('cadeado_acao',           'login'),
  ('cadeado_link',           '')
ON CONFLICT (chave) DO NOTHING;

-- 7. Trigger: cria perfil automaticamente quando usuário se cadastra
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, nome, email, role)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'nome', split_part(NEW.email, '@', 1)),
    NEW.email,
    'user'
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();

-- 8. Políticas de segurança (RLS)
ALTER TABLE profiles       ENABLE ROW LEVEL SECURITY;
ALTER TABLE licencas       ENABLE ROW LEVEL SECURITY;
ALTER TABLE progresso      ENABLE ROW LEVEL SECURITY;
ALTER TABLE convites       ENABLE ROW LEVEL SECURITY;
ALTER TABLE configuracoes  ENABLE ROW LEVEL SECURITY;

CREATE POLICY "profiles_select" ON profiles FOR SELECT USING (auth.uid() = id);
CREATE POLICY "profiles_update" ON profiles FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "profiles_super_select" ON profiles FOR SELECT
  USING (EXISTS (SELECT 1 FROM profiles p WHERE p.id = auth.uid() AND p.role = 'super'));
CREATE POLICY "licencas_select_user" ON licencas FOR SELECT
  USING (id IN (SELECT licenca_id FROM profiles WHERE id = auth.uid()));
CREATE POLICY "licencas_super_all" ON licencas FOR ALL
  USING (EXISTS (SELECT 1 FROM profiles p WHERE p.id = auth.uid() AND p.role = 'super'));
CREATE POLICY "licencas_admin_select" ON licencas FOR SELECT
  USING (licenca_admin_id = auth.uid());
CREATE POLICY "licencas_update_member" ON licencas FOR UPDATE
  USING (id IN (SELECT licenca_id FROM profiles WHERE id = auth.uid()));
CREATE POLICY "licencas_select_auth" ON licencas FOR SELECT
  USING (auth.role() = 'authenticated');
CREATE POLICY "progresso_all" ON progresso FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "progresso_super" ON progresso FOR SELECT
  USING (EXISTS (SELECT 1 FROM profiles p WHERE p.id = auth.uid() AND p.role = 'super'));
CREATE POLICY "convites_select" ON convites FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "convites_insert" ON convites FOR INSERT WITH CHECK (auth.role() = 'authenticated');
CREATE POLICY "convites_update" ON convites FOR UPDATE USING (auth.role() = 'authenticated');
CREATE POLICY "cfg_select" ON configuracoes FOR SELECT USING (true);
CREATE POLICY "cfg_insert" ON configuracoes FOR INSERT
  WITH CHECK (EXISTS (SELECT 1 FROM profiles p WHERE p.id = auth.uid() AND p.role = 'super'));
CREATE POLICY "cfg_update" ON configuracoes FOR UPDATE
  USING (EXISTS (SELECT 1 FROM profiles p WHERE p.id = auth.uid() AND p.role = 'super'));

-------- FIM DO SQL --------

SOLUÇÃO DE PROBLEMAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

404 ao acessar /login ou /admin:
  Verifique se Publish Directory no Render é "." (ponto).
  Os arquivos _redirects e render.yaml devem estar na raiz.

Login não funciona:
  Verifique SUPABASE_URL e SUPABASE_KEY em auth.js.
  Console do navegador (F12) mostra o erro exato.

Loop após login / não redireciona:
  Execute no Supabase SQL:
  SELECT * FROM profiles WHERE email = 'seu@email.com';
  Se vazio, o trigger não criou o perfil. Execute:
  INSERT INTO profiles (id, email, nome, role)
  SELECT id, email, raw_user_meta_data->>'nome', 'user'
  FROM auth.users WHERE email = 'seu@email.com';

Código inválido ao ativar:
  Verifique a tabela licencas. O campo ativa deve ser true.
  O código é convertido para maiúsculas automaticamente.

Deploy falhou no Render:
  Render → seu site → "Deploys" → clique no deploy → leia o log.
  Geralmente é o Publish Directory errado (use ".").
